## 1. 产品概述

### 1.1 产品定位

AI 装修效果图生成工具。用户上传户型图，选择装修风格，快速生成装修效果图。

### 1.2 核心用户故事

> 作为业主，我想上传户型图并选择装修风格，以便快速看到未来家的装修效果，省去反复沟通和等待设计师出图的时间。

### 1.3 MVP 范围

**做（In Scope）**
- 户型图上传（图片格式：JPG/PNG，最大 10MB）
- AI 户型结构识别（墙、门、窗、房间分区）
- 3 种装修风格生成（现代简约、北欧风格、现代中式）
- 效果图展示与下载

**不做（Out of Scope）**
- 3D 交互漫游
- 材质/家具更换
- 施工图导出
- 视频漫游
- 微信/小程序
- 付费订阅

## 4. 后端接口

### 4.1 接口列表

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/upload` | POST | 上传户型图 |
| `/api/recognize` | POST | 识别户型结构 |
| `/api/generate` | POST | 生成效果图 |
| `/api/result/{task_id}` | GET | 查询结果 |

详细接口规格见 `docs/API.md`。

### 4.2 任务流程

```
上传图片
  ↓
/api/recognize → 户型识别（YOLO + 墙线检测）
  ↓
存储识别结果（JSON），提交生成任务
  ↓
/api/generate → SD + ControlNet 生成效果图
  ↓
/api/result/{task_id} → 返回效果图 URL
```

### 4.3 错误处理

| 场景 | 处理方式 |
|------|----------|
| 图片尺寸过大 | 返回错误码 1001，提示用户压缩 |
| 图片格式不对 | 返回错误码 1002，提示支持的格式 |
| 户型识别失败 | 返回错误码 2001，提示用户上传清晰的户型图 |
| 生成失败 | 返回错误码 2002，显示默认兜底图 |
| GPU 排队中 | 客户端轮询，等待完成 |

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 3s（首屏） |
| 图片上传速度 | < 5s（10MB 文件） |
| 户型识别耗时 | < 5s |
| 效果图生成耗时 | < 15s |
| 支持并发用户 | 10 人同时在线 |

### 5.2 可用性要求

- 图片上传失败自动重试（最多 3 次）
- 生成失败显示友好提示 + 重试按钮
- 移动端基本可用（响应式布局）

### 5.3 数据存储

| 数据类型 | 存储方式 | 保留时间 |
|----------|----------|----------|
| 上传原图 | 腾讯云 COS | 7 天 |
| 效果图 | 腾讯云 COS | 7 天 |
| 识别结果 | 内存/Redis | 与任务绑定 |
| 访问日志 | 不存储 | - |

## 6. 验收标准

### 6.1 完成定义（DoD）

- [ ] 用户能上传 JPG/PNG 户型图（≤10MB）
- [ ] 上传后能预览原图
- [ ] 能看到 AI 识别出的墙体高亮显示
- [ ] 能选择 3 种风格之一
- [ ] 点击生成后 15s 内看到效果图
- [ ] 能下载效果图到本地
- [ ] 移动端页面可正常访问

### 6.2 测试用例

| 用例 | 步骤 | 期望结果 |
|------|------|----------|
| 正常上传 | 上传一张清晰的户型图 JPG | 上传成功，显示预览 |
| 图片过大 | 上传一张 15MB 的图 | 提示图片过大，请压缩 |
| 格式错误 | 上传一个 PDF 文件 | 提示格式不支持 |
| 风格生成 | 选择北欧风格，点击生成 | 显示 4 张北欧风格效果图 |
| 生成失败 | 用一张风景照片上传 | 提示识别失败，请上传户型图 |
